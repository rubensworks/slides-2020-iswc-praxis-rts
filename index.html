RDF specifications provide test suites

  • SPARQL, JSON-LD, Turtle, RDFa, ...

  • To prove specification compliance

    Using test reports in the EARL vocabulary

  • Test suites are declarative

      {
    "@context": ["context.jsonld", {"@base": "toRdf-manifest"}],
    "@id": "",
    "@type": "mf:Manifest",
    "name": "Transform JSON-LD to RDF",
    "sequence": [{
      "@id": "#t0001",
      "@type": ["jld:PositiveEvaluationTest", "jld:ToRDFTest"],
      "name": "Plain literal with URIs",
      "purpose": "Tests generation of a triple using full URIs and a plain literal.",
      "input": "toRdf/0001-in.jsonld",
      "expect": "toRdf/0001-out.nq" },
      ...
    ]
  }

Declarative test suites cause large overhead for developers

  1. Download the test manifest
  2. Parse the manifest based on a certain RDF syntax
  3. Follow links to other parts of the manifest if applicable
  4. Interpret the test definitions as executable logic
  5. Execute all tests
  6. Report the test results

RDF Test Suite alleviates this burden

  • Takes care of all steps to execute test suites

    Developer can focus on actual implementation of specification

  • Inspired by popular testing frameworks

    Fast repeated execution, visually appealing result formatting, test filtering, ...

  • Open-source, for JavaScript

    https://github.com/rubensworks/rdf-test-suite.js
